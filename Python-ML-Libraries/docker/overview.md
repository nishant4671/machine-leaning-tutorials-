# Docker Overview 🐳

"It works on my machine!" - Now it works everywhere.

### What is Docker?
Docker is a platform for developing, shipping, and running applications in "Containers." A container is a lightweight, standalone package that includes everything needed to run your code.

### Why use it?
It solves the "dependency hell." Your code will run exactly the same way on your laptop, your friend's Mac, and a Linux server in the cloud.

### Core Concepts
1. **Image**: A blueprint for your environment (the "DVD").
2. **Container**: A running instance of an image (the "Movie").
3. **Dockerfile**: A text file containing the instructions to build an image.

### Installation
Download **Docker Desktop** for Windows/Mac or install via terminal on Linux.

### Alternatives
- **Podman**: A daemonless alternative to Docker.
- **Conda Envs**: Good for local Python environments, but doesn't handle system-level dependencies.
