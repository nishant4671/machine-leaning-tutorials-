# 1. Functions + Type Hints

---

# Basic Function Syntax

```python id="cs56zh"
def add(a, b):
    return a + b
```

---

# Breaking the Syntax

```python id="ek3yl0"
def add(a, b):
```

| Part     | Meaning                |
| -------- | ---------------------- |
| `def`    | define/create function |
| `add`    | function name          |
| `(a, b)` | inputs/parameters      |
| `:`      | function body starts   |

---

# With Type Hints

```python id="1v23mo"
def add(a: int, b: int) -> int:
    return a + b
```

---

# Syntax Breakdown

| Part     | Meaning                  |
| -------- | ------------------------ |
| `a: int` | a should be integer      |
| `b: int` | b should be integer      |
| `-> int` | function returns integer |

---

# English Translation

```python id="p5r2xu"
def add(a: int, b: int) -> int:
```

Means:

> “Create a function named add.
> It accepts two integers
> and returns an integer.”

---

# Another Example

```python id="12af40"
def greet(name: str) -> str:
    return "Hello " + name
```

---

# Visual Mental Model

```text id="tt4a6s"
INPUT  ---> FUNCTION ---> OUTPUT

 "Nishant"    greet()     "Hello Nishant"
```

---

# Why Type Hints Matter

Without type hints:

```python id="y4t2bm"
def process(data):
```

Nobody knows:

* what data is,
* expected structure,
* expected return type.

---

# With Type Hints

```python id="uvwsfy"
def process(data: list[str]) -> int:
```

Now much clearer.

---

# Real ML Example

```python id="x5lqqn"
def predict(text: str) -> float:
    return 0.98
```

Meaning:

* input = text
* output = confidence score

# 2. Imports in Python

---

# What is an Import?

Import means:

> “Bring tools from another file/module.”

---

# Example

```python id="lrxw1q"
import math
```

Now you can use:

```python id="gdl0dr"
math.sqrt(25)
```

---

# Why Needed?

Python itself is small.

Most power comes from modules/libraries.

---

# Types of Imports

---

# A. Full Import

```python id="smq8jh"
import math
```

Usage:

```python id="7s9m92"
math.pi
```

---

# B. Specific Import

```python id="nmjlwm"
from math import pi
```

Now directly:

```python id="9bm0e4"
print(pi)
```

---

# C. Multiple Imports

```python id="vj67ha"
from math import pi, sqrt
```

---

# What is a Module?

A module is simply:

> a Python file containing useful code/tools.

Examples:

* math
* random
* os
* typing

---

# What is a Library?

Library = collection of modules.

Example:

* NumPy
* Pandas
* PyTorch

---

# 3. The `typing` Module

---

# What is `typing`?

Python module containing tools for type hints.

Example:

```python id="iqn9tw"
from typing import Optional
```

Meaning:

> “Bring Optional tool from typing module.”

---

# Important Type Hint Tools

| Tool             | Meaning          |
| ---------------- | ---------------- |
| `list[int]`      | list of integers |
| `dict[str, int]` | dictionary       |
| `Optional[str]`  | string OR None   |

---

# 4. Understanding `Optional`

---

# Core Idea

Sometimes a variable:

* may contain value,
* OR may contain nothing.

Python uses:

```python id="v9hgyh"
None
```

for “nothing.”

---

# Example

```python id="t7rj1t"
nickname = None
```

Later:

```python id="4d93w0"
nickname = "Nishu"
```

---

# Proper Type Hint

```python id="hcg91w"
from typing import Optional

nickname: Optional[str] = None
```

---

# Meaning

```python id="57zh6q"
Optional[str]
```

means:

```python id="4nt8q0"
str OR None
```

---

# Visual

```text id="7mx3dg"
nickname
   ├── "Nishu"
   └── None
```

---

# Real ML Example

```python id="wjlyy0"
model_path: Optional[str] = None
```

Maybe:

* model not loaded yet,
* path assigned later.

# 6. Dataclasses — Deep Notes

This is one of the MOST important Python production concepts.

---

# Problem Without Dataclasses

Using dictionaries:

```python id="uxm1v9"
student = {
    "name": "Nishant",
    "age": 18
}
```

Problems:

* typos,
* unclear structure,
* difficult maintenance,
* no autocomplete.

---

# Dataclass Solution

```python id="x5yrm6"
from dataclasses import dataclass
```

---

# Basic Syntax

```python id="njlwm6"
@dataclass
class Student:
    name: str
    age: int
```

---

# VERY IMPORTANT IDEA

This creates:

> structured blueprint for objects.

---

# What is `@dataclass`?

This is a:

# Decorator

Decorator means:

> “modify/enhance something automatically.”

---

# Here

```python id="q58v2g"
@dataclass
```

tells Python:

> “Automatically add useful functionality to this class.”

---

# Without Dataclass

You must manually write:

```python id="9pzk0r"
class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age
```

Very repetitive.

---

# Dataclass Generates This Automatically

Cleaner code:

```python id="hj3u7u"
@dataclass
class Student:
    name: str
    age: int
```

---

# Creating Objects

```python id="g3s8c1"
s1 = Student("Nishant", 18)
```

---

# Accessing Data

```python id="0r1ts7"
print(s1.name)
print(s1.age)
```

---

# 7. Dataclass Features

---

# A. Constructor Automatically Generated

Without dataclass:

```python id="jx3rjw"
def __init__(self, name, age):
```

must be written manually.

---

# With Dataclass

Python creates it automatically.

So:

```python id="26mx2v"
Student("Nishant", 18)
```

works immediately.

---

# B. Readable Printing

Without dataclass:

```python id="k1qz1l"
print(student)
```

Ugly result:

```text id="yvv2pb"
<__main__.Student object at 0x...>
```

---

# With Dataclass

```text id="nlyukw"
Student(name='Nishant', age=18)
```

Much cleaner.

---

# C. Equality Checks

Without dataclass:

```python id="cf2w2n"
student1 == student2
```

compares memory locations.

---

# With Dataclass

Compares actual field values.

---

# Example

```python id="rrk6js"
s1 = Student("Nishant", 18)
s2 = Student("Nishant", 18)

print(s1 == s2)
```

Output:

```python id="ynhhnn"
True
```

---

# D. Clean Structure

Everyone instantly understands:

* available fields,
* expected data types,
* object structure.

Huge benefit in large systems.

---

# 8. Real ML Dataclass Example

```python id="h52w5f"
from dataclasses import dataclass

@dataclass
class Prediction:
    text: str
    score: float
```

---

# Creating Prediction

```python id="9odk4o"
p = Prediction(
    text="positive",
    score=0.98
)
```

---

# Access

```python id="b31lfa"
print(p.text)
print(p.score)
```

---

# Why Dataclass Good Here?

Prediction objects:

* repeat often,
* have fixed structure,
* must remain clean/readable.

Perfect dataclass use case.

---

# 9. Advanced Dataclass Example

---

# Default Values

```python id="l1c56h"
@dataclass
class User:
    name: str
    age: int = 18
```

---

# Example

```python id="w7rmpy"
u = User("Nishant")
```

Age automatically becomes 18.

---

# Optional Fields

```python id="4mbyl8"
from typing import Optional

@dataclass
class User:
    nickname: Optional[str] = None
```

# 10. Important Production Python Habits

---

# Use Clear Names

BAD:

```python id="scmh5m"
x = 5
```

GOOD:

```python id="e0tcr6"
user_age = 5
```

---

# Use Type Hints

BAD:

```python id="i8h0qi"
def process(x):
```

GOOD:

```python id="m1pq1s"
def process(x: list[str]) -> int:
```

---

# Keep Functions Small

BAD:

Huge 300-line function.

GOOD:

Small focused functions.

---

# Organize Data Properly

Prefer:

```python id="6pv29u"
@dataclass
```

instead of messy dictionaries everywhere.
